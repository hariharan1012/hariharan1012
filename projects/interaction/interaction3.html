Project Overview

Problem Statement

The client, a leading American Bank Holding company, wanted a solution to monitor their Loan and Investment compliance application. Their existing application took the users in a elaborate round off manner to complete their task. This lead to loss of productivity. The client wanted a solution that would be a one-stop solution to monitor loan and investment and also enable the users to make edits and modifications if necessary.

My Approach

To understand the problem statement, my team and I had connect with the client and a walkthrough session on the existing application. We then performed a deeper analysis on the application to find the most optimal path for a user to achieve their task. After several discussions and ideation, we sketched our approach on our solution. Post getting a soft sign off from the client, we started our design phase.

The Design

As the users tasks were defined and pre-set, we requested access to sample data that would help us understand the magnitude of the application. Discovery Phase

The Workflow

We approached this phase, with emphasis more focus to the process along with the user needs. This decision was because, we understood that the user groups who would be using this application were defined with specific set of tasks. This made our process to understand the user journey a lot easier. Hence, we wanted to ensure that user was able to achieve their tasks with as less clicks as possible without impacting the performance of the application.

Learnings

DEFINE THE PROCESS

As old as the quote, the pen is sharper than the sword, we as designers need to consider user needs as a priority, but in the event where the user journey is defined, but the path is too long, we need to look at options to minimize that path. This does not eliminate the user needs, but a process optimally defined will aid the design to be minimalistic and simple.
